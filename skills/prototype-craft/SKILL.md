# prototype-craft — 프로토타입·슬라이드 품질 기준

## 목적
참가자 데모가 **한눈에 완성도 있어 보이게** 만든다. 기능 수보다 **정돈된 시각 위계·일관성·디테일**이 심사(데모 가능성)에서 점수를 좌우한다. `prototype-build`가 화면/슬라이드를 만들 때 이 기준을 적용한다.

## 0. 시작 전 — 디자인 브리프 (팀이 정하게)
만들기 전에 **팀의 그림을 먼저 끌어낸다**(개방형): *"어떤 느낌이면 좋겠어요? 참고하고 싶은 앱·색 있어요?"* 팀이 말한 걸 아래 항목으로 정리해 확인한다. 잘 안 나오면 그때 보기를 **보조로** 제시하고, "알아서 해줘"면 기본값으로. 이미 `context.md`/대화에 있으면 생략.
1. **무드/톤** — 미니멀·깔끔 / 친근·경쾌 / 포멀·신뢰 / 감성·따뜻 중? (기본: 주제 무드에 맞춤)
2. **메인 색** — 브랜드/선호 색 있나요? (기본: 주제 연상색 — 바다=청록, 산·숲=딥그린, 도심=인디고, 미식=테라코타, 역사=버건디)
3. **대상 기기** — 발표 스크린(가로) 중심 / 모바일 중심 / 둘 다? (기본: 둘 다 반응형)
4. (선택) **참고 스타일** — 좋아하는 앱·사이트 느낌이 있나요?
> 피칭덱을 만들 때도 동일하게 **톤·색·표지 문구**를 먼저 확인한다.
답을 `context.md`의 로그/아이디어 메모에 남겨 재사용한다(중복 질문 방지).

## 제약 (반드시 유지)
- **단일 HTML 파일**, 외부는 CDN만(폰트: Pretendard CDN 허용, 차트: Chart.js). **localStorage 금지**, 상태는 JS 변수.
- 오프라인 데모 가능해야 하므로 이미지는 **CSS 그라디언트·도형·이모지·인라인 SVG**로 대체(외부 이미지 URL 의존 금지).
- 한국어 UI, 모바일 우선 반응형.
- (설치돼 있으면) `theme-factory`·`brand-guidelines` 스킬로 테마·브랜드를 얹어도 좋다. 없으면 아래 기본 시스템을 쓴다.

## 1. 디자인 토큰 (기본 시스템)
`:root`에 변수로 선언해 일관성 유지.
- **색**: 브랜드 1색(primary) + 보조 1색(accent) + 잉크(#111~#1f2933) + 배경(#f6f8fa) + 뮤트(#6b7c88). 주제 무드에 맞춰 primary 선택(바다=청록, 산=딥그린, 도심=인디고, 미식=테라코타).
- **타이포**: 한 서체(Pretendard 권장). 스케일 1.25배수 — 12/14/16/20/26/34px. 본문 15~16px, 줄높이 1.5~1.6.
- **간격**: 4의 배수(4/8/12/16/24/32/48). 여백을 **넉넉히**(빽빽함이 아마추어 느낌의 1순위).
- **모서리·그림자**: radius 10~16px, 그림자는 은은하게 1단계만(`0 1px 6px rgba(0,40,70,.08)`).

## 2. 레이아웃 원칙
- **최대폭 컨테이너**(720~960px) 가운데 정렬. 전체폭 방치 금지.
- 명확한 **위계**: 히어로(핵심 메시지+핵심 수치) → 섹션별 h2 → 카드 그리드.
- **그리드**: `repeat(auto-fill,minmax(210px,1fr))`로 카드 반응형.
- 한 화면 = 한 목적. 스크롤로 흐름을 만든다.

## 3. 컴포넌트 패턴
- **히어로**: 그라디언트 배경 + 제목 + 한 줄 소개 + **핵심 지표 3~4개 스탯칩**(데이터로 신뢰 주기).
- **칩/필터**: 선택 상태 뚜렷(테두리→채움 전환).
- **결과 카드**: 배지(강조 라벨)·제목·태그·설명. 강조 대상은 좌측 컬러바나 골드 배지로 차별.
- **빈 상태**: "무엇을 하면 되는지" 안내 문구(빈 화면 방치 금지).
- **마이크로 인터랙션**: hover/active 트랜지션 .15s, 버튼 눌림 피드백. 과하지 않게.

## 4. 피칭덱 크래프트
- **1슬라이드 1메시지.** 제목은 문장형(주장), 본문은 3줄 이내.
- 고정 **슬라이드 프레임**: 좌상단 kicker(섹션명) + 큰 제목 + 콘텐츠 + 우하단 페이지·각주. 전 슬라이드 동일 골격 → 일관성.
- **데이터 슬라이드엔 차트 1개**(Chart.js) — 비교/분포를 시각화. 축·범례 색을 테마와 맞춤.
- 흐름: 표지 → 문제 → **데이터 근거** → 솔루션 → 데모 → AI 활용 → 임팩트/다음. (피칭 4요소: 문제근거·사용자가치·AI활용·데모)
- 키보드 ←/→ 네비, 페이지 인디케이터.

## 5. 폴리시 체크리스트 (산출 전 자가점검)
- [ ] 색은 3~4색 이내로 절제됐나(무지개 금지).
- [ ] 여백이 충분한가(요소가 서로 안 붙어 있나).
- [ ] 타이포 위계가 보이나(제목/본문/캡션 구분).
- [ ] 핵심 수치가 히어로/데이터슬라이드에 크게 보이나.
- [ ] hover/선택 등 인터랙션 피드백이 있나.
- [ ] 빈 상태·모바일 폭에서도 깨지지 않나.
- [ ] 외부 이미지·localStorage 없이 오프라인으로 뜨나.
- [ ] 슬라이드 골격이 전부 일관된가.

## 6. 안티패턴 (피하기)
빽빽한 여백, 5색 이상 남발, 기본 파랑 링크/기본 버튼 방치, 전체폭 텍스트 벽, 저해상 외부 이미지, 슬라이드마다 다른 레이아웃, 애니메이션 과다.

> 요약: **절제된 색 · 넉넉한 여백 · 뚜렷한 위계 · 큰 핵심 수치 · 일관된 슬라이드 골격.** 이 다섯이 데모 완성도의 90%.
